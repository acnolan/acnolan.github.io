A Chat about Chernoff Faces

Andrew Nolan

I started writing this blog post over a year ago. In fact, if I had finished it then, it would have been my first real blog post. That was my intention anyway. However, things happened, and now here we are many blog posts later. I think it is still an interesting topic and I hope you enjoy reading about it.

In the summer of 2020, I took the course CS 548, Knowledge Discovery and Data Mining, at WPI. In one of the lectures we talked about data visualizations and briefly, in one slide, in that one lecture, we discussed Chernoff Faces. For some reason I think about this data visualization very often, probably more than I should. So, I wanted to share them with you, hopefully you find them as fascinating as I do.

Plotting Multivariate Data

Plotting data effectively is hard. And most of the time we do it, we do it for two dimensions of data, an X and a Y axis. If you had to display three, or even more, dimensions of data in a graph how would you do it?

Some common options are extending a basic scatter plot into more dimensions. You can draw it in 3d or flatten it into scatterplot matrix.

Another popular design is to make an interactive parallel coordinates plot. These plots often let viewers rearrange the order of the axes to analyze the patterns in the data.

These are great (and popular) options. But you may already feel a little bit overwhelmed by these plots. How are you supposed to read and interpret them? That's the big challenge that data visualization scientists seek to solve. These are just some of the options.

Another Option, Chernoff Faces!

An alternative multivariable plot type is Chernoff Faces!

Chernoff Faces were proposed in 1973 by the American Mathematician, Herman Chernoff. These faces provide a way to visualize multivariable data by mapping data features to visual features on a face. Proponents of this visualization claim that humans can easily recognize faces and identify small changes between faces. Making these faces useful in pattern recognition, outlier detection, or clustering similar data objects.

On the flip side, the scale and specific values of the data are often obscured in these visages. Without intimate knowledge of which values are mapped to each facial feature it is hard to decipher anything beyond how much the data varies.

To better understand these plots, I think an example will be useful. For this example, I will write the code in MATLAB. If you are interested in exploring the code I used for yourself, you can find it here.

We are going to use the famous Iris Dataset. This is a famous dataset first used by statistician and biologist Ronald Fisher in the 1930s. If you have taken a machine learning class before, you have probably seen an example with it. It is a very simple multivariate dataset using a few measurements of Iris flowers.

The textbook question for this data is how can you use these values (sepal length, sepal width, petal length, and petal width) to predict which subspecies of Iris a sample belongs to?

Typically, this question will be answered using machine learning. But here we will try to answer it through the power of data visualization with Chernoff Faces.

First, we will plot the data related to each subspecies. The goal is to identify patterns based in the data based on how the faces are displayed.

We will map the data features to the following facial features:

  • Sepal Length -> Size of face
  • Sepal Width -> Forehead/jaw relative arc length
  • Petal Length -> Shape of forehead
  • Petal Width -> Shape of jaw

Some people may be able to, others may not. This is the challenge of Chernoff faces and multivariate data plotting in general. There is no perfect visualization. There are tradeoffs to all designs, for some cases, the benefits of Chernoff faces may be worth it. We will talk about that more in the next blog!
